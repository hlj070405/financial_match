# 会话记录功能实现说明

## 📋 功能概述

实现了完整的自动会话记录功能，用户在聊天时会话历史会自动保存到数据库，并在右侧边栏实时显示。

---

## 🔄 完整流程

### 1. 用户发送消息
```
用户输入消息 → 前端发送POST请求到 /api/chat
```

**前端代码位置**: `ChatModule.vue:550-605`
- 构建payload包含: `message`, `style`, `files`(可选), `conversation_id`(可选)
- 如果是新会话，不传`conversation_id`

```
后端接收请求 → 生成或使用conversation_id → 开始流式响应
```

**后端代码位置**: `main.py:287-289`

### 3. 流式响应开始
```
后端 → 发送'connected'事件 → 前端接收并保存conversation_id
```

**后端代码位置**: `main.py:308`
```python
yield f"data: {json.dumps({'type': 'connected', 'message': '已连接到服务器', 'conversation_id': conversation_id})}\n\n"
```

**前端代码位置**: `ChatModule.vue:636-641`
```javascript
if (parsed.type === 'connected') {
  if (parsed.conversation_id && !currentConversationId.value) {
    currentConversationId.value = parsed.conversation_id
  }

```

### 4. 收集AI响应
```
Dify返回text_chunk事件 → 后端累积到full_response → 前端显示
```

**后端代码位置**: `main.py:399-406`
```python
elif event == 'text_chunk':
    text = data.get('data', {}).get('text', '')
    if text:
        has_streamed_text = True
        full_response += text  # 累积完整响应
        yield f"data: {json.dumps({'type': 'text', 'text': text})}\n\n"
```

### 5. 自动保存会话
```
workflow_finished事件 → 后端自动保存 → 数据库存储
```

**后端代码位置**: `main.py:424-438`
```python
# 自动保存聊天历史
if request.save_history and full_response:
    try:
        await chat_history_service.save_or_update_chat(
            db=db,
            user_id=current_user.id,
            conversation_id=conversation_id,
            message=request.message,
            response=full_response,
            is_first_message=is_new_conversation
        )
        print(f"[自动保存] 会话历史保存成功: {conversation_id}")
    except Exception as save_error:
        print(f"[自动保存] 保存失败但不影响响应: {str(save_error)}")
```

### 6. 刷新历史列表
```
前端接收finish事件 → 调用loadChatHistory() → 更新右侧边栏
```

**前端代码位置**: `ChatModule.vue:649-663`
```javascript
else if (parsed.type === 'finish') {
    // ... 处理其他数据
    await loadChatHistory()  // 刷新历史列表
    console.log('会话已自动保存,历史列表已刷新')

```

## 🗄️ 数据库结构

### ChatHistory 表
```sql
CREATE TABLE chat_history (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT,                    -- 用户ID
    conversation_id VARCHAR(50),    -- 会话唯一标识
    title VARCHAR(200),             -- AI自动生成的标题
    first_message TEXT,             -- 首条用户消息
    messages TEXT,                  -- JSON格式的完整消息历史
    created_at DATETIME,            -- 创建时间
    updated_at DATETIME             -- 更新时间
);
```

### messages字段格式
```json
[
  {"role": "user", "content": "用户消息1"},
  {"role": "assistant", "content": "AI回复1"},
  {"role": "user", "content": "用户消息2"},
  {"role": "assistant", "content": "AI回复2"}
]
```

### 1. AI自动生成标题

```python
title = await self.deepseek_service.generate_chat_title(message, response)
```

- 使用DeepSeek API分析对话内容
- 生成简洁有意义的标题
- 仅在创建新会话时调用

### 2. 会话继续机制
**代码位置**: `chat_history_service.py:28-46`
```python
chat = await run_in_threadpool(
    lambda: db.query(ChatHistory).filter(ChatHistory.conversation_id == conversation_id).first()
)

if chat:
    # 更新现有会话
    messages = json.loads(chat.messages) if chat.messages else []
    messages.append({"role": "user", "content": message})
    messages.append({"role": "assistant", "content": response})
    chat.messages = json.dumps(messages, ensure_ascii=False)
```

### 3. 错误处理
- 保存失败不影响用户体验（静默失败）
- 使用try-except包裹保存逻辑
- 记录日志便于调试

## 🔧 API端点

### 1. 获取历史列表
```

      "conversation_id": "uuid",
      "title": "AI生成的标题",

      "created_at": "2024-01-01T00:00:00",
      "updated_at": "2024-01-01T00:00:00"

  ]
}
```

### 2. 获取会话详情
```
GET /api/chat/history/{conversation_id}
Authorization: Bearer {token}
```

## ⚠️ 注意事项

1. **save_history参数**: 默认为true，如果不想保存某些会话，前端可以传`save_history: false`

2. **conversation_id格式**: 使用UUID v4格式，确保全局唯一

3. **字符集**: 数据库使用utf8mb4，支持emoji和特殊字符

4. **并发问题**: 使用`run_in_threadpool`确保数据库操作不阻塞异步流

5. **错误恢复**: 即使保存失败，用户仍能看到完整的AI回复
