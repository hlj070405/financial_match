## Dify Workflow配置说明

### 工作流结构
```
Start (输入)
  ├─ input: 用户问题
  ├─ files: 上传的文件
  └─ style: 分析风格
      ↓
Document Extractor (文档提取)
  └─ 提取files中的文本内容
      ↓
LLM (财务分析)
  ├─ System Prompt: 财务分析意图识别
  ├─ Context: 文档提取的内容
  └─ User Input: input + style
      ↓
End (输出)
  └─ text: 分析结果
```

### 关键配置
- **模型**: Pro/moonshotai/Kimi-K2.5
- **Temperature**: 0.7
- **Response Mode**: blocking
- **文件上传限制**: 15MB, 最多20个文件

## 常见问题排查

### Q1: Dify连接失败
**错误信息**: "Dify工作流执行失败: Connection refused"

**解决方案:**
1. 检查Dify服务是否运行
2. 确认 `.env` 中的 `DIFY_API_URL` 配置正确
3. 测试Dify API: `curl http://localhost/v1/workflows/run`

### Q2: 文件上传失败
**错误信息**: "文件上传到Dify失败"

**解决方案:**
1. 检查文件大小是否超过15MB
2. 确认文件格式是否支持
3. 检查Dify API Key是否正确

### Q3: 认证失败
**错误信息**: "无法验证凭据"

**解决方案:**
1. 重新登录获取新token
2. 检查token是否过期
3. 确认后端JWT配置正确

### Q4: Workflow返回空结果
**错误信息**: "抱歉，我现在无法回答这个问题"

**解决方案:**
1. 检查Dify workflow是否正确配置
2. 查看Dify日志确认执行状态
3. 确认LLM节点的输出映射正确
