## 一、开发背景

### 1.1 行业痛点

随着我国资本市场的不断发展，A股上市公司已超过5000家，每年产生数万份财务报告、研究报告和公告文件。个人投资者和中小金融机构面临以下核心痛点：

- **信息过载**：单份企业年报动辄数百页，人工阅读效率极低，关键数据难以快速定位
- **分析门槛高**：量化策略回测通常需要编程基础，普通投资者无法自主验证交易想法
- **数据分散**：行情数据、财务数据、新闻资讯分布在不同平台，缺乏统一的分析入口
- **工具碎片化**：现有方案要么侧重文档问答（如ChatPDF），要么侧重行情展示（如同花顺），缺少从"阅读理解→策略验证→模拟交易"的完整闭环

### 1.2 技术背景

近年来大语言模型（LLM）和检索增强生成（RAG）技术的突破，为解决上述问题提供了新的技术路径：

- **RAG技术**使LLM能够基于用户私有文档进行精准问答，避免幻觉（hallucination）
- **Agent Function Calling**使LLM能够自主调用外部工具，实现多步骤复杂任务的自动化编排
- **向量数据库**（如ChromaDB）的成熟，使大规模文档的语义检索成为可能
- **开源量化框架**（如Backtrader）降低了策略回测的工程门槛

### 1.3 项目目标

本系统旨在打造一个**AI驱动的一站式金融分析决策支持平台**，让普通用户通过自然语言对话即可完成：

1. 上传财报PDF → 自动解析入库 → 智能问答分析
2. 用自然语言描述交易策略 → 自动执行历史回测 → 获取专业量化报告
3. 实时查看行情数据 → AI联网搜索最新新闻 → 生成综合分析报告
4. 模拟买卖操作 → 自动计算A股真实费率 → 跟踪持仓盈亏

## 二、软件特点

### 2.1 全链路AI Agent架构

系统以**Kimi k2.5大语言模型**为核心大脑，采用多轮Function Calling架构。Agent可自主决策调用6个专业工具：

| 工具名称 | 功能说明 |
| --- | --- |
| search_vector_store | 在用户知识库中进行语义检索 |
| fetch_financial_report | 从巨潮资讯网自动抓取财报PDF并解析入库 |
| run_backtest | 将自然语言策略翻译为条件JSON，执行量化回测 |
| portfolio_trade | 记录虚拟买卖操作，模拟A股交易费率 |
| portfolio_query | 查询虚拟持仓和交易记录 |
| $web_search | Kimi内置联网搜索，获取实时新闻和市场动态 |

Agent在一次对话中可多次调用不同工具，实现复杂任务的自动拆解与执行。例如用户说"帮我分析宁德时代最新财报，顺便用MACD策略回测一下"，Agent会依次调用财报抓取→入库→检索→回测→生成综合报告。

### 2.2 三阶段RAG检索漏斗

传统RAG仅依赖向量相似度，存在"实体混淆"问题。本系统设计了三阶段检索漏斗：

```
阶段1：意图解析 + 元数据过滤 + 向量召回
  ↓ Top-K候选文档
阶段2：BM25词频评分 + RRF融合重排
  ↓ 精排候选
阶段3：BGE Cross-Encoder精排 + 断路器
  ↓ 最终结果（或触发联网补充）
```

- **阶段1**：从查询中提取公司名、年份、季度，构建ChromaDB元数据where条件，缩小检索范围
- **阶段2**：自研轻量BM25实现，与向量分数通过RRF（Reciprocal Rank Fusion）融合，提升硬实体命中率
- **阶段3**：调用BGE-Reranker-v2-m3做query-document整体注意力评分；若最高分低于阈值，触发断路器自动切换联网搜索

### 2.3 自然语言→量化策略翻译

用户无需编程，用自然语言描述策略即可执行回测：

- 支持**11种技术指标**：price, sma(N), ema(N), rsi(N), macd(), macd_signal(), boll_upper(N), boll_lower(N), boll_mid(N), volume, volume_ma(N)
- 支持**6种运算符**：`>`, `<`, `>=`, `<=`, `cross_above`, `cross_below`
- 策略条件格式：`{"left": "sma(5)", "op": "cross_above", "right": "sma(20)"}`
- FlexibleStrategy类动态创建Backtrader指标对象，逐K线求值并执行交易

### 2.4 智能文档分块

针对财报PDF的结构化挑战，系统实现了Markdown感知的智能分块引擎：

- 使用pymupdf4llm将PDF转为结构化Markdown，保留标题层级和表格格式
- 章节标题评分函数通过编号前缀、长度、标点、数字占比等多维度识别真实标题
- 表格整体保留策略：检测到表格行特征后不切分，超长表格截断并标注
- 每个chunk注入上下文前缀（当前章节路径），确保脱离上下文后仍可理解

### 2.5 流式响应与实时反馈

全系统采用**SSE（Server-Sent Events）流式输出**：

- Agent对话过程中，用户可实时看到思考过程和工具调用进展
- 事件分类：`phase`（阶段提示）、`delta`（文本增量）、`tool_start/tool_result`（工具过程）、`done`（完成）
- 行情分析服务支持Redis互斥锁 + 后台异步执行 + 前端轮询，防止重复分析

## 三、软件优势

### 3.1 与同类产品对比

| 能力维度 | ChatPDF类产品 | 传统行情软件 | 量化平台 | **本系统** |
| --- | --- | --- | --- | --- |
| 财报文档问答 | ✅ 基础RAG | ❌ | ❌ | ✅ 三阶段漏斗RAG |
| 实时行情展示 | ❌ | ✅ | ⚠️ 部分 | ✅ K线+技术指标+资金流向 |
| 联网新闻搜索 | ❌ | ⚠️ 基础 | ❌ | ✅ Kimi $web_search |
| 量化策略回测 | ❌ | ❌ | ✅ 需编程 | ✅ 自然语言驱动 |
| 模拟持仓交易 | ❌ | ⚠️ 部分 | ⚠️ 部分 | ✅ A股真实费率模拟 |
| 多工具智能编排 | ❌ | ❌ | ❌ | ✅ Agent自主决策 |
| 全链路闭环 | ❌ | ❌ | ❌ | ✅ 阅读→分析→回测→交易 |

### 3.2 核心技术优势

**（1）检索精度领先**

三阶段漏斗 + 断路器机制，有效解决向量检索的"实体混淆"问题。查询路由层先提取意图并构建元数据过滤，BM25补充词频匹配，Cross-Encoder做最终精排，不相关内容被断路器拦截并自动切换联网补充。

**（2）零编程量化回测**

用户说"5日均线上穿20日均线买入"，Agent自动翻译为标准化条件JSON并调用Backtrader引擎。回测环境模拟A股真实费率（佣金万2.5、卖出印花税千0.5、100股整手限制），返回收益率、夏普比率、最大回撤、胜率等专业指标和资金曲线图。

**（3）一站式数据整合**

系统打通了Tushare Pro（行情/财务/资金流向）、BaoStock（历史日线）、巨潮资讯网（财报下载）、Kimi联网搜索（实时新闻）四大数据源，通过Redis分级缓存策略（static/daily/quarterly）实现高效数据服务，避免重复API调用。

**（4）生产级工程质量**

- Docker Compose一键部署5个服务（MySQL + Redis + ChromaDB + 后端 + 前端）
- 数据库迁移（Alembic）、JWT认证、CORS跨域、健康检查
- Redis分布式锁防并发冲突，指数退避重试处理LLM过载
- 防脏数据机制：API返回空或error时不覆盖缓存，旧数据继续服务

### 3.3 用户体验优势

- **角色化设计**：提供"大众用户""高净值用户""专业投资者"三种角色预设，按需加载功能模块
- **对话式交互**：所有功能通过自然语言对话触发，无需学习复杂操作界面
- **过程透明**：Agent工具调用过程实时展示，用户可看到AI的"思考过程"
- **多格式导出**：对话内容支持导出为PDF，分析报告支持Markdown渲染

## 五、功能模块

### 5.1 智能咨询模块（Agent对话）

本模块是系统的核心交互入口，基于Kimi k2.5的多轮Function Calling实现。

**主要功能：**

- **RAG智能问答**：基于用户上传的财报文档进行深度问答，支持多文档对比分析。回答附带来源溯源（文档名、页码、相关度分数）
- **财报自动抓取**：用户说"帮我看比亚迪2024年年报"，Agent自动从巨潮资讯网下载PDF→解析→入库→检索→生成分析报告
- **量化策略回测**：用户描述交易策略，Agent翻译为条件JSON调用回测引擎，返回收益率、夏普比率、最大回撤等指标和资金曲线
- **虚拟持仓操作**：用户说"帮我买入1000股比亚迪"，Agent记录交易并自动计算佣金和印花税
- **联网实时搜索**：通过Kimi $web_search获取最新新闻、政策、市场动态，补充知识库覆盖不到的信息
- **文件工作台**：管理用户已上传的文档，支持在线预览PDF、选择性分析

**技术实现：**

- Agent循环最多15轮，每轮处理LLM返回的tool_calls
- 支持工具调用碎片拼装（流式返回时参数跨多个chunk）
- SSE实时推送思考过程和工具执行进度
- 会话历史管理（创建、重命名、删除、自动生成标题）

### 5.2 行情终端模块

集成Tushare Pro数据源，提供专业级行情分析界面。

**主要功能：**

- **自选股管理**：添加/删除自选股，一键切换查看
- **K线与基础行情**：日K线/周K线/月K线，开高低收成交量
- **技术指标叠加**：MACD、RSI、布林带等经典技术指标可视化
- **资金流向分析**：主力资金、散户资金净流入/流出趋势
- **AI智能分析**：对选中股票调用Kimi+联网搜索，生成SSE流式综合分析报告（基本面+技术面+资讯面）

**技术实现：**

- Tushare Pro提供日线、周线、月线、每日指标（PE/PB/换手率）、资产负债表、现金流量表等接口
- Redis持久化缓存层（非传统TTL过期），分三级更新策略：
  - static（永不失效）：股票基本信息、交易日历
  - daily（每日更新）：日线行情、资金流向
  - quarterly（季度更新）：财务报表数据
- 防脏数据机制：API返回空或error时不覆盖，旧数据继续服务
- ECharts渲染K线图、技术指标叠加图、资金流向图

### 5.3 智能检索模块

提供独立的语义搜索界面，让用户直接检索知识库内容。

**主要功能：**

- **语义搜索**：自然语言输入查询，返回最相关的文档片段
- **相关度分数**：每个结果显示向量相似度分数
- **来源溯源**：显示文档名称、页码位置
- **文件类型过滤**：按PDF、Excel、Word等类型筛选
- **排序选项**：按分数、页码、来源名称排序

**技术实现：**

- 调用三阶段检索漏斗（向量召回→BM25重排→Cross-Encoder精排）
- 每用户独立ChromaDB collection（user_{id}_docs），数据隔离
- 支持阈值过滤，低于最低分数的结果自动剔除

### 5.4 量化回测模块

提供独立的可视化回测界面，也可通过Agent对话触发。

**主要功能：**

- **股票选择**：输入股票代码选择回测标的
- **策略选择**：提供预置策略（SMA交叉、MACD、RSI超买超卖）和自定义条件策略
- **参数配置**：回测时间范围、初始资金、佣金费率
- **结果展示**：
  - 汇总统计：总收益率、年化收益率、夏普比率、最大回撤、胜率、交易次数
  - ECharts资金曲线图
  - 交易明细列表（买入/卖出时间、价格、盈亏）

**技术实现：**

- Backtrader引擎封装，自定义A股佣金模型（AShareCommission）
  - 佣金：万2.5（双向），最低5元
  - 印花税：千0.5（仅卖出）
  - 整手限制：100股整数倍（AShareRoundLotSizer）
- BaoStock获取历史日线数据，Parquet本地缓存避免重复请求
- FlexibleStrategy支持用户自定义买卖条件，动态创建技术指标
- 分析器集成：TradeCollector（交易明细）、EquityCurve（资金曲线）、SharpeRatio、DrawDown

### 5.5 数据源管理模块

管理系统的文档知识库，支持多种方式导入数据。

**主要功能：**

- **单文件上传**：拖拽或选择PDF文件上传，自动解析、分块、向量化入库
- **批量导入**：多文件一键批量上传与处理
- **文档列表**：查看已入库文档的名称、页数、分块数、入库时间
- **文档删除**：删除文档同时清理ChromaDB中的向量和MySQL中的元信息

**技术实现：**

- 上传文件存储在 `financial_reports/user_{id}/` 目录，按用户隔离
- PDF解析优先使用pymupdf4llm（结构化Markdown），fallback到pdfplumber（表格提取）
- 分块策略：MarkdownHeaderTextSplitter（标题层级切分）→RecursiveCharacterTextSplitter（超长段落二次切分）
- Embedding批量处理（batch_size=32），通过SiliconFlow API调用bge-large-zh-v1.5
- Document元信息（公司名、股票代码、年份）存储在MySQL，便于管理和查询

### 5.6 虚拟持仓模块

通过Agent对话触发，模拟真实A股交易环境。

**主要功能：**

- **买入操作**：记录买入股票、价格、数量，自动计算佣金
- **卖出操作**：记录卖出，自动计算佣金+印花税，更新持仓成本
- **持仓查询**：查看当前持有股票列表、持仓数量、平均成本、市值、盈亏
- **交易流水**：查看历史交易记录

**技术实现：**

- 采用"分为单位"存储（1元=100分），避免浮点数精度问题
- 加权平均成本法计算持仓成本
- A股费率模型：
  - 佣金：成交金额 × 万2.5，最低5元
  - 印花税：成交金额 × 千0.5，仅卖出收取
- 数据模型：Portfolio（投资组合）+ PortfolioTransaction（交易流水），支持多组合

### 5.7 用户认证模块

**主要功能：**

- 用户注册/登录，JWT Token认证
- 角色选择：大众用户、高净值用户、专业投资者
- 功能模块按角色配置，用户可自定义启用/禁用模块
- Token有效期30天，自动续期

**技术实现：**

- python-jose生成JWT Token
- passlib + bcrypt密码哈希
- FastAPI Depends依赖注入实现路由级鉴权
- 前端localStorage持久化用户配置
