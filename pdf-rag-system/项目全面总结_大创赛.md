# 幻流 · 智能金融分析平台（FinFlow）

> 基于大语言模型与检索增强生成（RAG）技术的一站式金融信息智能分析系统

---

## 一、项目定位

面向个人投资者、金融从业者及财经研究人员，提供 **"数据采集 → 智能解析 → 深度分析 → 辅助决策"** 全链路金融信息服务。系统通过 RAG 技术将非结构化财报文档转化为可检索的知识库，结合多源实时数据与 LLM 推理能力，实现对上市公司财务健康度、行业竞争格局和市场风险的智能化、结构化分析。

---

## 三、已完成核心功能模块

### 1. RAG 智能问答引擎（核心亮点）

**技术路线**：PDF 解析 → Markdown 结构化切分 → bge 向量化 → ChromaDB 索引 → 混合检索 + BGE Rerank → LLM 流式生成

- **多引擎 PDF 解析**：pymupdf4llm 优先转 Markdown 保留结构，PyMuPDF + pdfplumber 兜底处理扫描件与复杂表格
- **智能分块**：基于 Markdown 标题层级的逻辑分块（5120 token/块），保留页码、表格等元数据
- **意图路由**：自动识别用户查询意图（定位检索 / 汇总分析 / 对比），动态构建过滤条件
- **混合重排序**：语义相似度 + BGE Rerank 二阶段精排，提升检索精准度
- **每用户独立知识库**：`user_{id}_docs` 隔离 Collection，支持多用户并行使用

### 2. AI 智能对话

- **多轮流式对话**：基于 SSE 的实时流式输出，支持思维过程可视化
- **文档工作台**：上传 PDF 绑定到会话，限定 RAG 检索范围，实现精准文档问答
- **会话管理**：自动保存历史、LLM 智能生成会话标题、支持续接/重命名/删除
- **角色适配**：支持"专业分析""通俗解读"等多种输出风格

### 3. 实时行情与数据中心

- **A 股行情**：日线/分钟线 K 线图、实时报价、自选股管理（Tushare）
- **宏观经济**：CPI、GDP 趋势图（AKShare）
- **行业板块**：板块排名、涨跌幅对比
- **股票搜索**：在线 API 搜索 + 本地数据库模糊匹配

### 4. Agent 智能分析

- **联网搜索分析**：Kimi k2.5 内置 `$web_search`，一轮对话自动联网获取最新资讯并生成分析
- **Redis 互斥锁**：防止同一股票重复分析，结果缓存 10 分钟
- **异步执行**：后台线程异步运行，前端轮询状态

### 5. 全景运营诊断

- **财务健康诊断**：LLM 驱动的结构化 JSON 分析，覆盖盈利能力、偿债能力、运营效率等维度
- **多维对标分析**：双公司横向对比，输出雷达图数据
- **风险评估**：经营风险、财务风险、市场风险分级量化
- **全部 SSE 流式输出**，前端实时渲染分析过程

### 6. 产业链分析

- **产业链图谱**：LLM 生成上下游产业链结构，Vue Flow 交互式可视化
- **竞争格局**：行业内主要参与者、市占率、护城河分析
- **供应链风险**：关键供应商依赖度、替代风险评估

### 7. 舆情与热点追踪

- **多源聚合**：新浪财经、华尔街见闻、同花顺、财联社四大数据源实时爬取
- **关键词搜索**：支持按数据源过滤、按关键词检索
- **缓存策略**：Redis 缓存避免高频爬取

### 8. 财报自动获取

- **LLM 意图识别**：自然语言输入 → AI 解析股票代码 + 报告期
- **巨潮资讯网爬虫**：自动从巨潮下载对应 PDF 财报
- **自动入库**：下载完成后自动触发 RAG 向量化入库

## 四、规划中功能模块

以下功能为下一阶段重点迭代方向，已完成技术方案设计，部分已有原型验证。

### 规划 1：企业数据库接入

**为什么要做这个功能**

当前系统的数据来源以 PDF 财报和公开行情 API 为主，而企业内部数据库中沉淀了大量高价值的业务数据（如 ERP 系统、CRM 数据、内部经营报表），这些数据无法通过公开渠道获取。打通企业数据库接入，意味着系统可以**从"公开数据分析工具"升级为"企业级数据中台"**，覆盖更广泛的分析场景。

**与现有功能的对接方案**

- **数据层对接**：用户在前端配置数据库连接串（MySQL / PostgreSQL / Oracle），后端通过 SQLAlchemy 动态创建连接引擎，复用现有 ORM 架构
- **RAG 融合**：将数据库表中的关键业务数据序列化为自然语言文本，走现有 RAG Pipeline（分块 → 向量化 → ChromaDB 入库），使用户可以用自然语言查询企业内部数据
- **前端对接**：在现有"数据源管理"模块中新增"数据库接入"Tab 页（已有 `ds_upload / ds_batch / ds_multi` 的 Tab 架构），提供连接配置、表预览、一键入库等功能
- **元数据标记**：ChromaDB 中通过 `source_type: "database"` 区分数据库来源与 PDF 来源，检索时可按来源类型过滤

### 规划 2：Agent 分析信号回测验证（AI 可靠性量化证明）

**为什么要做这个功能**

当前系统的分析链路是**开环**的：Agent 给出"看多/看空/风险评级"等分析结论后，用户只能凭主观判断来采纳或忽略，缺乏客观的可靠性评估。我们计划接入聚宽（JoinQuant）等量化回测平台，**将 Agent 的分析结论本身作为 alpha 因子进行历史回测**，用真实市场数据验证 AI 分析的预测能力。这在方法论层面实现了从"AI 给结论"到"AI 结论可量化验证、可持续迭代"的质变，将系统从"信息工具"升级为**"可证伪的决策验证系统"**。

**核心思路**

不同于传统的因子回测（验证 ROE、PE 等财务指标的有效性），本方案验证的是**AI Agent 分析能力本身**：

```
历史数据回放：对过去 N 只股票 × M 个时间点
    │
    ▼
Agent 逐个分析 → 输出结构化信号（评分/看多看空/置信度）
    │
    ▼
以 Agent 信号作为因子 → 构建多空组合
    │
    ▼
回测收益曲线、夏普比率、IC/IR 值、分行业胜率
    │
    ▼
结论示例：
  "Agent 分析信号 IC 均值 0.05，月度 ICIR 1.8"
  "对周期股判断准确率 72%，对成长股仅 48%"
  "结合 RAG 检索深度 >3 篇财报时，信号质量显著提升"
```

若回测发现 Agent 在特定场景下不靠谱，可反向驱动优化：调整 prompt 策略、增加检索深度、引入更多数据源——形成**"分析 → 验证 → 迭代"的科学闭环**。

**与现有功能的对接方案**

- **Agent 层对接**：复用现有 Kimi k2.5 Function Calling 架构（`agent/service.py`），批量调用 Agent 对历史时间点进行分析，收集结构化信号（评分、方向、置信度）
- **诊断模块对接**：现有 `diagnosis/` 模块的财务诊断、风险评估均输出结构化 JSON，可直接解析为回测信号
- **数据层对接**：复用现有 Tushare 行情数据获取股票历史收益率，聚宽 SDK（`jqdatasdk`）执行组合回测与绩效归因
- **前端对接**：在诊断分析模块中增加"信号回测"结果卡片，使用 ECharts 渲染累计收益曲线、IC 序列、分行业胜率热力图等
- **缓存策略**：回测结果通过现有 Redis 缓存体系存储，相同回测配置短期内不重复计算

### 规划 3：云端数据工作台（分析成果自动沉淀）

**为什么要做这个功能**

用户在使用系统过程中会产生大量分析结果（诊断报告、对标分析、风险评估、回测数据等），但目前这些结果散落在各个会话中，缺乏统一的结构化归档。云端数据工作台让 Agent 每次分析完成后，自动将结构化结果写入用户专属的在线表格，实现**分析成果的持续积累与复用**。

**与现有功能的对接方案**

- **存储层**：服务器端为每个用户维护一个 Excel 文件（`user_exports/{user_id}/analysis_workbook.xlsx`），通过 `openpyxl` 追加写入
- **Agent 对接**：现有诊断分析（`diagnosis/`）、产业链分析（`chain_api.py`）、Agent 分析（`agent/service.py`）的返回结果均为结构化 JSON，解析后直接映射为 Excel 行列数据
- **前端对接**：新增"数据工作台"页面，使用 Naive UI 的 `n-data-table` 在线预览表格内容，支持筛选、排序和一键下载
- **并发安全**：通过文件锁或异步队列保证多个 Agent 任务同时写入时的数据一致性
- **与 PDF 导出互补**：现有 `html2pdf.js` 导出的是可视化报告，Excel 工作台导出的是原始结构化数据，满足不同使用场景

## 五、系统特色总结

| 亮点 | 说明 |
|------|------|
| **端到端 RAG 自研** | 从 PDF 解析到向量检索到流式问答全链路自主实现，非调用第三方 RAG SaaS |
| **多模型协同** | Kimi k2.5（推理 + 联网）+ DeepSeek（意图）+ bge（Embedding）+ Qwen（辅助），按任务分配最优模型 |
| **全流式体验** | 所有 LLM 交互均采用 SSE 流式输出，用户感知延迟极低 |
| **多源数据融合** | 结构化行情数据（Tushare/AKShare）+ 非结构化文档（PDF 财报）+ 实时资讯（新闻爬虫），未来扩展企业数据库 |
| **决策闭环（规划中）** | AI 分析 → 量化回测验证 → 因子迭代优化，从信息工具升级为决策验证系统 |
| **容器化一键部署** | Docker Compose 编排 5 个服务，环境变量统一配置，开箱即用 |
| **金融级可视化** | ECharts K 线图、Vue Flow 产业链图谱、雷达图对标分析，数据即图表 |
